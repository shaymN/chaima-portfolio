Project 2 – Router Web UI & Configuration Flows

Context

I validated the router Web UI used by customers to configure their home network: Wi-Fi, LAN, firewall, guest network, and parental control.

Main Areas Tested

  • Login & authentication (admin credentials, password change).
  • Wi-Fi setup: SSID, password, security mode, hide/visible network.
  • Guest Wi-Fi network and isolation from main LAN.
  • Device list and parental control rules.
  • Port forwarding, firewall, and remote access settings.
  • Diagnostics pages (ping, traceroute, logs).

Test Approach

  • Designed UI test cases covering all configuration flows and edge cases.
  • Tested across different browsers and devices (laptop, tablet, mobile).
  • Combined structured test cases with exploratory testing sessions.
  • Validated that UI feedback (success, error, warnings) is clear and helpful.

Example Test Scenarios

  • Change Wi-Fi password and verify that all devices must reconnect.
  • Create a guest Wi-Fi network and ensure it cannot reach main LAN devices.
  • Block a device using parental control and confirm traffic is denied.
  • Open a port with port forwarding and verify external access works as expected.
  • Use diagnostics tools (ping, traceroute) and confirm output correctness.

Defect Examples

  • UI showing “saved successfully” but settings not applied on the device.
  • Blocked devices still having internet access via guest network.
  • Incorrect error message when entering special characters in fields.

Outcome

This testing improved the consistency of the router UI, reduced customer confusion, and helped ensure that what the user sees and configures actually matches the real device behaviour.
