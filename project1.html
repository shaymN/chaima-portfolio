Project 1 – Full CPE Firmware Validation (DOCSIS/GPON)

Context

As a CPE Test Analyst, I worked on validating new firmware releases for DOCSIS/GPON routers used by large European ISPs. The goal was to ensure that real customer scenarios work reliably after each firmware upgrade.

Scope & Focus Areas

  • Customer scenarios (main focus): internet browsing, streaming, gaming, video calls, parental control, device blocking/allowing.
  • Wi-Fi behaviour: 2.4/5 GHz, channel changes, throughput, roaming, stability over time.
  • Diagnostics & system: logs, LEDs, reboot/factory reset behaviour, error messages.

Example Customer Scenarios

  • Family home: parents working from home + kids streaming/gaming on Wi-Fi.
  • Guest network usage with restricted access.
  • Parental control with scheduled internet cut-off (bedtime rules).
  • Blocking specific devices and verifying that traffic is actually blocked.

Wi-Fi & Performance Testing

  • Measured Wi-Fi throughput on 2.4 GHz and 5 GHz using iPerf and IxChariot.
  • Validated channel selection and impact of interference (neighbour networks).
  • Ran long-duration stability tests (overnight sessions under load).
  • Verified roaming behaviour when moving between rooms.

Diagnostics & System Validation

  • Checked system logs for errors or warnings after stressful test runs.
  • Verified LED behaviour (internet down, Wi-Fi off, WPS active, etc.).
  • Tested reboot and factory reset flows, including impact on saved settings.
  • Ensured that UI messages are clear and aligned with the real device status.

Tools & Technologies

  • DOCSIS / GPON
  • Wireshark, iPerf, IxChariot
  • Router Web UI, TR-069, logs

Impact

By focusing on realistic customer scenarios and Wi-Fi performance, we caught several issues before deployment (Wi-Fi instability after long uptime, incorrect parental control behaviour). This reduced field incidents and improved customer satisfaction.
