Project 3 – TR-069 ACS & Remote Management Testing

Context

I participated in testing the TR-069 integration between the router (CPE) and the ACS, making sure that remote management commands and parameter updates worked reliably and safely.

Key Areas

  • Get/SetParameterValues for Wi-Fi, WAN and diagnostics parameters.
  • Remote reboot and factory reset commands.
  • Fault handling and error reporting.
  • Security aspects (authentication, SSL, certificates).

Sample Test Cases

  • Read current Wi-Fi SSID and verify that it matches the router UI.
  • Change Wi-Fi password via ACS and confirm that clients must reconnect.
  • Trigger remote reboot and check that the device comes back online correctly.
  • Set invalid parameter values and validate that the device returns proper faults.

Tools & Techniques

  • TR-069 messages analysis, XML inspection.
  • Wireshark for traffic inspection.
  • Router UI comparison for cross-checking parameter values.

Result

Thanks to this testing, we identified several issues with parameter consistency, faulty error handling, and edge cases in remote reboot/reset behaviour, which were fixed before rollout.
